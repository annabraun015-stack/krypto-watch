## 🎯 Einleitung: Die Wächter der Dezentralität

In einem System ohne Chef ist die Wahrheit das Ergebnis eines globalen Abgleichs. Ein **Bitcoin Node** (Knotenpunkt) ist der physische Anker dieser Wahrheit. Während Miner die Geschichte *vorschlagen*, sind es die Nodes, die sie *genehmigen*. 

Im Jahr 2026, in einer Welt zunehmender finanzieller Überwachung und algorithmischer Kontrolle, ist das Betreiben einer eigenen Node der ultimative Akt digitaler Selbstverteidigung. Es geht nicht nur darum, "Teil des Netzwerks" zu sein — es geht darum, die Spielregeln von Bitcoin (wie die 21 Millionen Obergrenze) für sich selbst physisch durchzusetzen. In diesem Guide dekonstruieren wir die Anatomie eines Knotens und erklären, warum "Don't trust, verify" nur mit eigener Hardware möglich ist.

## Sektion 1: Definition — Was ist ein 'Full Node'?

Eine Full Node ist ein Wirtschaftsprüfer, der niemals schläft und niemals bestochen werden kann.

<DefinitionBlock term="Autarke Validierung" icon="🖥️">
  Eine **Full Node** ist ein Computer, der das gesamte Bitcoin-Protokoll ausführt. Er lädt jeden Block seit dem Genesis-Block (2009) herunter und prüft autonom jede einzelne Signatur, jede Gebühr und jedes UTXO gegen das unveränderliche Regelwerk. Im Gegensatz zu 'Light Clients' vertraut eine Full Node niemandem — sie berechnet die gesamte Geschichte der Weltwährung von Grund auf neu.

## Sektion 4: Pruning vs. Archival — Speicher-Strategien

Muss man wirklich die gesamte Geschichte speichern, um sicher zu sein?
- **Archival Node:** Speichert jeden Byte seit 2009. Benötigt im Jahr 2026 ca. 1-2 Terabyte SSD-Speicher. Essenziell für Block-Explorer und Forscher.
- **Pruned Node:** Validiert alles wie eine Archival Node, löscht aber alte Blöcke, nachdem sie geprüft wurden. Man behält nur das aktuelle 'UTXO-Set'. Dies ermöglicht volle Souveränität auf Geräten mit wenig Speicherplatz (z.B. 50 GB).
- **SPV (Simple Payment Verification):** 'Light Clients', die nur Block-Header prüfen. Sie sind bequem als App, aber sie hängen von der Ehrlichkeit fremder Nodes ab.

## Sektion 5: Privatsphäre — Tor, I2P & IP-Leaks

Warum ist eine Node der beste Schutz für Ihre Identität?
Wenn Sie eine kommerzielle Wallet-App nutzen, erfährt der Anbieter Ihre IP-Adresse und all Ihre Guthaben. Eine eigene Node über **Tor (The Onion Router)** maskiert Ihre Herkunft komplett.
- **Keine Abfragen an Dritte:** Ihre Wallet fragt nur Ihre eigene Node. Niemand sonst weiß, für welche Adressen Sie sich interessieren.
- **Sovereign Stack:** Systeme wie 'Umbrel' oder 'RaspiBlitz' ermöglichen es, eine Node als privaten Server für Lightning, Dokumente und Passwörter zu nutzen.

## Sektion 7: FAQ — Experten-Wissen für Node-Betreiber

### 1. Kostet das Betreiben einer Node viel Strom?
Nein. Eine moderne Node auf einem Raspberry Pi oder Mini-PC verbraucht etwa so viel Energie wie eine LED-Glühbirne (5-10 Watt). Die Kosten liegen bei wenigen Euro pro Jahr — ein kleiner Preis für absolute finanzielle Freiheit.

### 2. Was passiert, wenn 90% der Miner die Regeln ändern wollen?
Die Miner können eine eigene Kette starten (Hard Fork), aber sie können *Ihr* Bitcoin-Netzwerk nicht ändern. Wenn Ihre Node die neuen Blöcke ablehnt, bleiben Sie auf der ursprünglichen Kette. Dies passierte im 'Blocksize War' 2017: Die Nodes gewannen gegen die Miner-Giganten.

### 3. Brauche ich eine statische IP-Adresse?
Nein. Dank Tor und modernen Protokollen kann eine Node hinter jedem Heimrouter betrieben werden, ohne Ports manuell öffnen zu müssen. Die Node findet ihre Peers vollautomatisch.

Das Betreiben einer Node ist der Übergang vom passiven Nutzer zum aktiven Wächter. Im Jahr 2026 ist klar: Bitcoin gehört denen, die es verifizieren. Eine eigene Node gewährt Ihnen Privatsphäre gegenüber Konzernen, Sicherheit gegenüber Angreifern und Souveränität gegenüber Staaten.
